## 🎯 전략 방향 (확정)

### 타겟 사용자
1. **중대형 클럽 (50~150명)** — 최우선, 운영진 다수, 회비 엄격 관리
2. 소규모 친목 동호회 (10~25명) — 부차적
3. 상업 체육관/강습 — 장기

### 포지셔닝
> **"50~150명 배드민턴 클럽이 네이버밴드 + 카톡방 + 엑셀 + 수금앱을
> 전부 버리고 이것 하나로 돌릴 수 있는 통합 운영 도구."**

### "완벽"의 정의
1. **기능의 넓이** — 모든 운영 도구 대체 가능 (최우선)
2. **자동화 수준** — 운영자가 안 해도 돌아가게
3. 기능의 깊이 — 각 영역의 세련도 (3순위)

### 결정된 설계
- **소통 범위**: 게시판 + 앨범 + 공지 (채팅은 카톡 유지)
- **알림 방식**: Web Push (PWA) + 이메일 이중화
- **레벨 시스템**: 배린이 퀴즈 ↔ 클럽 연동, 공개/비공개 **사용자 토글**
- **마이그레이션**: 엑셀 임포트 (회원 + 회비 중심), **고정 템플릿 방식**
- **게시판 구조**: 카테고리 분리 (자유/질문/후기/장터)

## 🔑 핵심 설계 결정사항

### 엑셀 임포트
- **방식**: 고정 템플릿 (다운로드 → 채우기 → 업로드)
- **라이브러리**: xlsx (SheetJS)
- **처리 위치**: 클라이언트 파싱 + 서버 액션 저장
- **duplicate 처리**: 전화번호 기반 매칭
- **placeholder user**: `users.birdieminton_user_id = null` 허용, 실제 가입 시 전화번호로 자동 매칭 후 merge

### 레벨 배지
- **저장**: `users` 테이블에 `quiz_level` (enum: 왕초보/초심자/D조/C조) + `show_level` boolean
- **표시**: 회원 프로필, 멤버 리스트, 경기 배정 시 참고

### 게시판
- **카테고리**: 자유(free), 질문(question), 후기(review), 장터(marketplace)
- **댓글**: 2단계 (댓글 + 대댓글)
- **좋아요**: 단순 하트 카운터

### PWA 알림
- **우선순위**: Web Push (PWA 설치자)
- **Fallback**: 이메일 (Push 구독 없으면)
- **향후**: 카카오 알림톡 (유료화 후)
